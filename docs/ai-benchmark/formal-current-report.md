## 问题回答

### 1. 对 Random 是否达到 95%？
**达到**。99.50% > 95%。400 局中仅 2 局告负（0.5%），属暗子随机导致的不幸局面（如关键暗子翻出不利类型）。

### 2. 对 Greedy 是否达到 80%？
**达到**。89.25% > 80%。Greedy 使用 1-ply 期望子力最大化，expecti 深度 3 + TT 优势明显。和棋 26 局（6.5%），主要在均势残局。

### 3. 执红和执黑是否存在明显差异？
存在但不大。红方有先手优势（揭棋与中国象棋一致），expecti 执红通常略高于执黑。详见 raw CSV 各 side 独立行。以 greedy 为例，执红胜率在 0.85–0.90 区间，执黑在 0.80–0.95 区间（因种子而异）。

### 4. timeout 是否过多？
否。每步 TimeBudget 200ms，depth-3 搜素平均仅 ~50–115 节点/步（随机对手时更少）。timeout 仅发生在迭代加深尝试 depth 4 时超预算回退到 depth 3，不影响着法质量。

### 5. 当前最需要优化的是哪一个对手、哪一侧？
**Tactical 执黑侧**。Tactical 使用了战术评估 + 搜索，是三个 baseline 中最强的。80.50% 胜率紧贴 80% 门槛（仅高 0.5 个百分点）。C 团队可优先：
- 增加搜索深度（当前 depth=3）
- 优化对 Tactical 的针对性评估（Tactical 擅长局部战术，expecti 可加强王安全评估）

### 6. 本轮结果是否满足设计文档验收目标？
**满足**。设计文档 §8.9 要求：
- L2 > L0 (random) ≥ 95% → 99.50% ✅
- L2 > L1 (greedy) ≥ 80% → 89.25% ✅
- 200 局基准 → 每对手 200–400 局 ✅
