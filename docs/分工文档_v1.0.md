## 1. 分工总原则（先立规矩，再分活）

1. **模块所有权制**：每个 Maven 模块有且只有一名 Owner。Owner 对该模块的代码质量、测试、
   文档负最终责任；别人可以提 PR，但合入由 Owner 审。避免"三个人改一个文件"式混乱。
2. **接口契约先行**：跨人协作只认三样东西——**判例测试**（规则契约）、**协议金样例**
   （消息格式契约）、**Agent/PlayerView 接口**（AI 契约）。契约先冻结再各自开工，
   谁改契约谁负责通知全组并改齐所有受影响方。
3. **结对互备（bus factor ≥ 2）**：每人有一个"副责模块"，每周至少评审副责模块一次 PR，
   保证任何人临时缺位时副责能顶上。
4. **完成定义（DoD）四条**：编译通过 → 相关测试全绿 → 一名指定评审人通过 → 相应文档段落已更新。
   四条不齐不算完成，不进主干。
5. **贡献可见**：所有提交用本人 git 账号；每个模块 README 署 Owner；报告各章署名。
   这既是课程个人成绩依据，也是相互监督机制。

## 4. 六张职责卡（每人的"从 0 到 1"）

### A —— 组长 & 规则引擎

- **使命**：全组只有一份规则真相，就在你手里。任何"这步到底合不合法"的争论以你的
  判例测试为终审；对外（老师、他组）的规则分歧由你裁定并留档。
- **交付物**：jieqi-common、jieqi-rules 全量实现；附录 B 判例测试 100% 绿 + 覆盖率报告；
  复盘重放器核心（棋谱 → 规则引擎重放，Q10）；Q&A 裁定台账（新增答疑 24h 内落到测试）。
- **验收标准**：`mvn test -pl jieqi-common,jieqi-rules -am`(或根目录 `mvn test`)全绿；任一组员摆出一个局面 + 着法，
  你能在 1 分钟内用一条测试给出可复现裁定。
- **组长事务**（约占 30% 工时，其余给代码）：周计划拆解与看板维护、里程碑门禁检查、
  争议仲裁、与老师的沟通窗口。
- **关键依赖**：无上游。**全组都依赖你**——所以规则模块必须最先完成、最先冻结。

### B —— 服务器

- **使命**：裁判执行器。规则由 A 定，执行、计时、广播、记录由你保证一毫不差，
  且他组任何奇怪报文都弄不崩你。
- **交付物**：登录/注册/匹配/先手协商；走子流水线（§6.4 九步，顺序即代码）；65s 计时与
  超时判负；差异化广播（吃暗子：吃方见真值、被吃方见 "NULL"）；>1KB 帧断连；棋谱双格式落盘。
- **验收标准**：§10.3 十二个集成场景全绿；两个客户端从登录到 gameOver 全流程无人工干预
  跑通；fuzz 一夜不崩（F 提供弹药）。
- **关键依赖**：A 的规则引擎（P0 末冻结）；向 D 提供可连的最小服务器（第 1 周末即需可登录+匹配，
  哪怕走子还没通——D 的 AiClient 骨架等这个）。

### C —— AI 算法

- **使命**：作业三条硬要求（暗子期望值 / Alpha-Beta / 概率赋值）全部落在你模块里，
  这是本组选题的得分核心。
- **交付物**：PlayerView（信息集棋盘，只从协议消息更新，杜绝偷看）；BeliefState 双池模型
  （含"你比对手更了解对手口袋"的不对称性，§8.2）；Evaluator（子力+期望+位置+王安全）；
  Expecti-Alpha-Beta 搜索（机会节点、落地必分支/离场折算、迭代加深）。
- **验收标准**：深度 3 最小版能稳定击败 L0 随机（>95%）；全量版对 L1 贪心 >80%；
  单步用时不超预算（D 的 harness 出数据）。
- **关键依赖**：A 的走法生成（直接复用 jieqi-rules，不自己写第二份）；D 的 harness
  （你的每次改动要当天拿到胜率反馈——没有评测的调参是玄学）。

### D —— AI 工程与评测

- **使命**：让 C 的算法能上场、不超时、不掉线，并用数据证明它有多强。
  组间赛的"不败底线"（§8.7）由你守。
- **交付物**：AiClient（连接/登录/自动匹配/断线重连/**拒着重发**三级容错）；TimeManager
  （预算 10s、硬上限 55s）；L0 RandomAgent、L1 GreedyAgent；SelfPlayHarness
  （一键起服务器+双 AI，跑 N 局出 CSV）；调参记录与胜率曲线；期望-实际方差日志（Q24 选做顺手拿）。
- **验收标准**：harness 连跑 200 局零崩溃、零超时判负；每次参数变更有 CSV 留档；
  L2>L0 95% / L2>L1 80% 两道门禁数据齐。
- **关键依赖**：B 的最小服务器（第 1 周末）；C 的 Agent 接口（第 1 周内冻结签名）。

### E —— 客户端

- **使命**：老师验收和真人对弈时看到的一切。功能正确优先于好看（Q43 明示无需美化），
  但棋盘、暗子、翻子结果、被吃"?"这些信息必须一眼可读。
- **交付物**：先 CLI 后 GUI 两形态——第 1 周先交 CLI（联调救急用，键入 `b2e2` 即走子）；
  第 2 周 Swing GUI 可完整对局（点击走子、暗子渲染、计时条、终局弹窗）；复盘查看器
  （加载棋谱逐步回放，复用 A 的重放核心）；对他组服务器的降级兼容（无 capturedPiece 显示"?"等，§4.9）。
- **验收标准**：两个 GUI 真人完整对局（P1 验收项）；连他组服务器不崩、字段缺失可降级；
  复盘器能回放任一棋谱文件。
- **关键依赖**：B 的协议实现；A 的规则引擎（本地合法性预检，减少 valid=false 往返）。

### F —— QA · 交付 · 文档

- **使命**：把"我们做完了"变成"我们能证明做完了"。测试证据、联调结果、交付包、
  报告——评分实际发生的地方都归你统筹。
- **交付物**：协议金样例测试（§4.2/4.3 全消息往返）；破坏性输入与 fuzz；12 个集成场景
  CI 化；组间联调 16 条检查清单（§10.4）的执行与记录（现场主谈人）；Docker 镜像 +
  start.sh/stop.sh + 共容器部署预案（Q32 加分项）；实验报告统稿、答辩 PPT、演示脚本。
- **验收标准**：任何人 clone 仓库后 `mvn test` 一条命令全绿；`docker compose up` 一键
  演示 AI 对弈；报告每一条结论都能指到测试输出/CSV/棋谱文件。
- **关键依赖**：所有人（你是下游汇聚点）——所以你有权在集成日打回不符合 DoD 的提交。

## 6. 协作机制

- **例会**：每周两次 15 分钟站会（各自：昨天/今天/卡点）；**周日集成日**（2h）：合分支、
  跑全量测试、过门禁、下周拆卡。
- **Git**：main 受保护；feature 分支 + PR；评审人 = 所有权矩阵里该项的 C 位成员；
  合入即触发全量 `mvn test`（哪怕没有 CI 服务器，集成日人肉跑也必须留输出存档）。
- **契约变更流程**：改判例测试/协议字段/Agent 接口 → 先发全组群 + 看板置顶卡 →
  受影响方确认 → 同一 PR 内改齐所有调用点。
- **决策记录**：口头争论不算数；结论进设计文档 §14 台账或 Q&A 台账，一句话 + 日期 + 依据。

TT 优化 → GUI 降级为 CLI。**永不砍**：规则正确性、协议兼容性、65s 不超时、棋谱记录。
