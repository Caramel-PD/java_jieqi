## 核心思路：不依赖网络的进程内测试

要测的是服务器**功能逻辑**：收到某条消息后，是否返回正确的响应、是否正确更新了双方状态。
这些逻辑全在 `ProtocolServer` 类中，跟 WebSocket 网络传输没有关系。

服务器代码已经把传输层抽象成了 `Core.ClientChannel` 接口：

```java
// Core.java:54
public interface ClientChannel {
    void send(String json);       // 服务器向客户端发消息
    void closeConnection();       // 服务器主动断连
    boolean isOpen();             // 连接是否存活
    String remote();              // 对端描述（日志用）
}
```

`ProtocolServer` 只依赖这个接口。正常运行时传入 WebSocket 实现；测试时传入一个假的实现
`FakeChannel`——`send()` 不经过网络，只把消息放进队列，测试代码再从队列里取出来断言。

相当于把网络层"短路"掉，在同一个 Java 进程内驱动服务器的全部协议逻辑。

## 基础设施：FakeChannel

`ProtocolServerTest.java` 里已经有一个可用的 `FakeChannel`（`private static final class`），
后续新增测试可以直接复用，或提取为独立的测试工具类。现有实现核心 API：

```java
FakeChannel ch = new FakeChannel("client1");
server.onConnected(ch);
server.onMessage(ch, jsonString);

ch.last();                          // 最近一条消息（String）
ch.lastOfType("moveResult");        // 最近一条指定 messageType（已解析 JsonObject）
ch.messagesOfType("moveResult");    // 全部指定 messageType 消息
ch.clear();                         // 清空 outbox
```

> 这个 API 比队列轮询更简洁——消息是同步送达的（同一线程内调用 `dispatch`），
> 所以 `last()` / `lastOfType()` 不需要等待，直接取即可。

## 端到端流程（对照清单场景）

下面以一个完整对局为主线，逐步说明每个清单场景怎么写测试。每一步标注了对应清单中的场景编号。

### 第 1 步：启动服务端 & 建立连接

对应清单场景：**2（Login）、3（register）**

```java
Core.ServerConfig config = new Core.ServerConfig();
config.autoRegisterOnLogin = true;   // 未注册用户登录时自动创建
config.usersFile = null;             // 不落盘，纯内存
ProtocolServer server = new ProtocolServer(config);

FakeChannel chA = new FakeChannel("A");
FakeChannel chB = new FakeChannel("B");
server.onConnected(chA);
server.onConnected(chB);
```

### 第 2 步：注册 / 登录

对应清单场景：**2（Login 成功）、3（register 成功）**

```java
// A 注册
server.onMessage(chA,
    "{\"messageType\":\"register\",\"userId\":\"u1\",\"password\":\"123\",\"nickname\":\"选手A\"}");
JsonObject r = chA.lastOfType("loginResult");
assertTrue(r.get("success").getAsBoolean());
assertEquals("u1", r.get("userId").getAsString());

// B 登录（autoRegisterOnLogin=true 自动创建账号）
server.onMessage(chB,
    "{\"messageType\":\"Login\",\"userId\":\"u2\",\"password\":\"123\"}");
r = chB.lastOfType("loginResult");
assertTrue(r.get("success").getAsBoolean());

chA.clear(); chB.clear();
```

### 第 3 步：匹配

对应清单场景：**5（startMatch → matchSuccess）**

```java
server.onMessage(chA, "{\"messageType\":\"startMatch\"}");
// A 进入等待队列，暂无回复
assertTrue(chA.outbox.isEmpty());

server.onMessage(chB, "{\"messageType\":\"startMatch\"}");
// 触发配对 → 双方各收 matchSuccess
JsonObject mA = chA.lastOfType("matchSuccess");
JsonObject mB = chB.lastOfType("matchSuccess");
assertEquals(mA.get("roomId").getAsString(), mB.get("roomId").getAsString());
assertEquals("u2", mA.get("opponentId").getAsString());
assertEquals("u1", mB.get("opponentId").getAsString());

chA.clear(); chB.clear();
```

### 第 4 步：Ready → gameStart

对应清单场景：**6（双 Ready → gameStart）、7（gameStart 字段校验）**

```java
server.onMessage(chA, "{\"messageType\":\"Ready\"}");
JsonObject infoB = chB.lastOfType("roomInfo");
assertTrue(infoB.get("opponentReady").getAsBoolean());

server.onMessage(chB, "{\"messageType\":\"Ready\"}");
JsonObject infoA = chA.lastOfType("roomInfo");
assertTrue(infoA.get("opponentReady").getAsBoolean());

// 双方就绪 → 各收定制版 gameStart
JsonObject gsA = chA.lastOfType("gameStart");
JsonObject gsB = chB.lastOfType("gameStart");
assertEquals("red",  gsA.get("yourColor").getAsString());
assertEquals("black", gsB.get("yourColor").getAsString());
assertTrue(gsA.get("firstHand").getAsBoolean());
assertFalse(gsB.get("firstHand").getAsBoolean());
assertEquals(32, gsA.getAsJsonArray("initialBoard").size());

chA.clear(); chB.clear();
```

### 第 5 步：走一步合法棋

对应清单场景：**8（合法 move）、9（flipResult 双发真值）**

```java
// 红方（A）首步暗子走子
server.onMessage(chA,
    "{\"messageType\":\"move\",\"fromX\":\"b\",\"fromY\":2,\"toX\":\"e\",\"toY\":2,\"isFlip\":true}");

JsonObject mrA = chA.lastOfType("moveResult");
JsonObject mrB = chB.lastOfType("moveResult");
assertTrue(mrA.get("valid").getAsBoolean());
assertTrue(mrB.get("valid").getAsBoolean());

// 首步暗子 → 双方 flipResult 相同（Q9）
assertTrue(mrA.has("flipResult"));
assertEquals(mrA.get("flipResult").getAsString(),
             mrB.get("flipResult").getAsString());

// move 字段原样
JsonObject move = mrA.getAsJsonObject("move");
assertEquals("b", move.get("fromX").getAsString());
assertEquals(2,   move.get("fromY").getAsInt());
assertEquals("e", move.get("toX").getAsString());
assertEquals(2,   move.get("toY").getAsInt());

chA.clear(); chB.clear();
```

### 第 6 步：非法走子

对应清单场景：**11（非法 move → valid=false）、12（非当前方 move）、13（from==to 拒绝）**

```java
// from==to → 2001（场景 13）
server.onMessage(chA, move("a", 0, "a", 0, true));
assertFalse(chA.lastOfType("moveResult").get("valid").getAsBoolean());
assertEquals(2001, chA.lastOfType("error").get("code").getAsInt());
assertTrue(chB.messagesOfType("moveResult").isEmpty()); // 对方无消息

chA.clear();
// 非当前方走子 → 2002（场景 12）：合法着法走后轮到黑方，红方再走即错
server.onMessage(chA, move("b", 2, "e", 2, false));
assertFalse(chA.lastOfType("moveResult").get("valid").getAsBoolean());
assertEquals(2002, chA.lastOfType("error").get("code").getAsInt());
```

### 第 7 步：终局 — Resign

对应清单场景：**15（Resign → gameOver）**

```java
// 任一方认输
server.onMessage(chA, "{\"messageType\":\"Resign\"}");

JsonObject goA = chA.lastOfType("gameOver");
JsonObject goB = chB.lastOfType("gameOver");
assertEquals("resign", goA.get("reason").getAsString());
assertEquals("black", goA.get("winner").getAsString());  // B 胜
assertEquals("resign", goB.get("reason").getAsString());

// 终局后 move 被忽略（场景 18）：不回复，不崩溃
assertDoesNotThrow(() -> server.onMessage(chA, move("b", 2, "e", 2, true)));
```
