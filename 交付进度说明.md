## 二、P0 已交付内容(可直接使用的地基)

### 2.1 jieqi-common(7 个类)—— 协议无关的公共领域类型

坐标系、棋子/颜色编码、着法、终局原因、错误码、JSON 宽容解析工具。编码与 JSON 名严格对齐
设计文档 §2.3 与公共接口 §4.6。

### 2.2 jieqi-rules(9 个类)—— 规则引擎(全组的"规则真相")

- **已实现且测试覆盖**:`RuleEngine`(走法生成/合法性校验/照面检测/将军检测/吃王判据)、
  `RepetitionTracker`(长将/长捉/80 半步状态机)、`BoardSnapshot`(不可变棋盘+机械落子)、
  `BoardText`(FEN 式局面文本,判例测试的局面描述语言)、`InitialLayout`(暗子虚拟类型唯一权威)、
  `ZobristHash`(局面哈希)、`CellState`/`Legality`/`RepetitionVerdict`。
- 全部纯函数、无网络、无线程、无随机——服务器裁判与 AI 走法生成将共用同一套,杜绝自我矛盾
  (设计 §3.4-4 规则单点原则)。

## 三、如何验证这份地基(拿到就能自测)

```bash
# 需 JDK 17+ 与 Maven;首次联网拉取 JUnit 5 依赖
# 重要:多模块工程,-am 表示"同时构建被依赖的 jieqi-common",否则会报找不到 jieqi-common
mvn test -pl jieqi-common,jieqi-rules -am

# 或直接在根目录跑(reactor 自动按依赖顺序 common->rules):
mvn test

# 期望输出:Tests run: 65, Failures: 0, Errors: 0, Skipped: 0
```

无 Maven 环境时的最小验证路径见 README(javac 直编 + JUnit standalone jar)。
本交付在容器内已用该路径实测通过。
